    F1 2077


    2 Mois : Février 2024 - Avril 2024


    “F1 2077” est un projet qui a été réalisé durant le 4e semestre du BUT Informatique ayant pour objectif de valider les compétences en JavaScript vues avec la ressource R4.A.10 mais aussi les compétences en gestion de projets vues avec la ressource R4.09. Ce projet comporte un jeu jouable depuis un navigateur développé en HTML, CSS et en JavaScript nommé “F1 2077” puisque ce jeu mélange les univers de la F1 et du jeu Cyberpunk 2077. Lorsque l’on arrive sur la fenêtre du jeu il faut d’abord se connecter à un compte qui existe déjà ou en crée un. Ensuite, on retrouve un menu proposant différentes options comme lancer une partie ou regarder le tableau des meilleurs scores du jeu. Si vous cliquez sur le bouton “Lancer une partie”, vous arriverez sur une fenêtre vous proposant de choisir un “skin” parmi ceux disponibles, vous aurez également 2 boutons : un bouton “Continuer” pour lancer la partie et un bouton “Retour” pour revenir au menu précédent. Vous pouvez jouer avec une 2e personne si cette dernière rejoint votre partie en utilisant le même URL que vous. Une fois la partie terminée (lorsque vous n’avez plus de vie), une fenêtre va apparaître sur votre écran vous permettant de relancer une partie ou revenir au menu du jeu. Durant la partie vous pouvez récupérer des bonus vous permettant d’obtenir des avantages. Le but du jeu est donc de rester en vie le plus longtemps et d’éliminer le plus d’ennemis possible. Mes tâches sur ce projet ont été la réalisation de la page “Crédits” et de la page de sélection de l’avatar.


      Compétences techniques acquises :
    • Utilisation du langage JavaScript
    • Utilisation de NodeJS
    • Utilisation de WebPack
    • Utilisation de Babel

      Savoir-être acquis :
    • Réalisation d'une tâche en un temps donné
    • Répartition des tâches dans un groupe
    • Communication entre différents membres d'un groupe

    Pizzaland


    1 Mois : Février 2024 - Mars 2024


    “Pizzaland” est un projet réalisé durant le 4e semestre du BUT Informatique. Ce projet utilise l’architecture REST qui comprend un contrôleur où l’on y retrouve les différentes méthodes permettant de réaliser différents types de requêtes JSON (méthode doGet permettant de réaliser des requêtes GET en JSON). Ensuite, nous avons plusieurs DTOs qui vont permettre de définir les caractéristiques d’un ingrédient ou d’une pizza par exemple dans le cas de cette SAE. Pour finir avec l’architecture REST, nous avons différents DAOs comprenant des méthodes comme findAll retournant l’ensemble des ingrédients. Les requêtes JSON permettant la réalisation de différentes actions ont été réalisé depuis un terminal. Une base de données Postgresql ainsi que le langage psql ont également été utilisés pour stocker les différentes pizzas ou ingrédients par exemple. L’application finale créée doit nous permettre de pouvoir gérer une pizzeria. Le but de ce projet était d’être capable de pouvoir créer et gérer un service REST à partir de ce qui fut vu en cours ou lors des Tps. Mes tâches dans ce projet ont été la réalisation complète de cette application car j’ai travaillé en autonomie sur ce projet.


      Compétences techniques acquises :
    • Utilisation du langage psql
    • Utilisation du logiciel Postgresql
    • Création de méthodes permettant la réalisation de requêtes JSON avec le langage Java
    • Création et utilisation d’un service REST
    • Création et utilisation de requêtes en JSON

      Savoir-être acquis :
    • Réalisation d'une tâche en un temps donné
    • Travail en autonomie
    • Réalisation d'un planning de travail

    Chasse au Monstre


    3 Mois : Octobre 2023 - Janvier 2024


    "Chasse au Monstre" est un projet réalisé durant le 3e Semestre du BUT Informatique ayant pour objectif de valider les compétences apprises dans les matières suivantes : Analyse, Qualité de développement, Développement efficace et Communication. Le projet « Chasse au Monstre » est un jeu utilisant le langage Java, dont le but est de trouver la sortie du labyrinthe sans se faire trouver par le chasseur lorsque l’on joue le monstre ou de trouver où est le monstre avant qu’il trouve la sortie du labyrinthe lorsque l’on joue le chasseur. Lorsque le jeu est lancé, vous pouvez jouer une partie ou paramétrez-le jeu. Si vous choisissez de jouer, vous pourrez choisir entre 3 modes de jeux, le 1er mode vous permettra de jouer contre un ami, le 2e mode de jouer contre une IA et le 3e de voir les IA s’affronter entre elles, une fois le mode choisi, vous choisirez votre personnage (sauf si vous avez choisi le 3e mode) puis la partie se lancera, les personnages jouent chacun leur tour en commençant par le monstre, si le monstre trouve la sortie ou que le chasseur trouve le monstre, la partie se termine, vous pourrez alors soit recommencer, soit revenir au menu. Si vous choisissez de paramétrer le jeu, vous pourrez alors modifier la couleur des différentes cases du labyrinthe ou encore augmenter la présence de murs ou de bonus par exemple.
    Dans ce projet, les tâches qui m'ont été confiés ont été la réalisation des différents menus du jeu, la modification et la refactorisation de certaines méthodes ou programmes Java, la création de documentation Java ainsi que la modification de fichiers FXML.


      Compétences techniques acquises :
    • Utilisation de SceneBuilder
    • Refactorisation d'un programme Java
    • Gestion de la taille des fenêtres JavaFX
    • Utilisation de FXML
    • Création de JavaDoc
    • Application d'idées et amélioration de solutions existantes

      Savoir-être acquis :
    • Réalisation d'une tâche en un temps donné
    • Répartition des tâches dans un groupe
    • Communication entre différents membres d'un groupe

    Mini-projet Agile


    3 jours : 4 Septembre 2023 - 7 Septembre 2023


    "Mini-projet Agile" est le premier projet du 3e Semestre et de la 2e année du BUT Informatique qui fut réalisé. Nous avons élaboré un jeu utilisant le langage Java se déroulant dans un donjon comprenant différents étages qui sont des niveaux où l'on retrouve différents types de monstres (du plus faible, facile à battre au plus fort nécessitant un certain inventaire pour être battu) à affronter pour passer au niveau, étage suivant. Entre les différents étages/niveaux se trouve une boutique permettant au joueur d'améliorer son inventaire en achetant de meilleures armes ou également de récupérer de la vie en achetant une potion. Le but du jeu est d'obtenir le score le plus élevé possible en battant le plus de monstre et en traversant le plus d'étages/niveaux possibles dans le donjon.
    Pour la réalisation de ce jeu, nous avons suivi la méthode Agile.
    Dans ce projet, les tâches qui m'ont été confiés sont la gestion du système de sauvegarde du jeu avec la création d'un système permettant d'enregistrer son score effectué et d'un système permettant de reprendre une partie arrêtée/mise en pause, ainsi que la création d'un mécanisme dont le but est d'effectuer un classement des dix meilleurs scores enregistrés.


      Compétences techniques acquises :
    • Planification du travail a effectué
    • Répartition du travail dans un groupe
    • Etre capable d'améliorer un projet
    • Création d'un système de sauvegarde en Java
    • Création d'un système de chargement de partie en Java
    • Création d'un système de classement en Java

      Savoir-être acquis :
    • Communication/Travail dans un groupe
    • Travail en autonomie en un temps donné
    • Echange avec membres d'un groupe

    Microsoft mobilité
